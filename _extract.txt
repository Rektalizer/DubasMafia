


    
    Полное ТЗ: Telegram-бот DUB Quest Game

    
    1. Общая идея

    Нужно создать Telegram-бота для группового чата.

    Бот превращает обычный чат в ежедневную игру с тайными заданиями, внутренней валютой DUB, кошельками, лидербордом, угадыванием чужих заданий и магазином игровых действий.

    Каждый день участники получают тайные квесты, выбирают один, выполняют его в общем чате и пытаются не быть раскрытыми. Остальные игроки наблюдают за поведением друг друга и могут попытаться угадать, какое задание выполняет конкретный игрок.

    DUB — это только внутренняя игровая валюта. Реальных денег, ставок, депозитов, вывода средств и финансовой ценности нет.

    

    
    
    
    2. Главная игровая логика

    Ежедневный цикл:

    В 08:00 по Москве бот отправляет каждому активному игроку в личку 3 тайных квеста на выбор:

    лёгкий;

    средний;

    тяжёлый.

    Игрок выбирает один квест кнопкой.

    После выбора бот пишет в общий чат:

    “🎭 @username принял секретный квест. Теперь его можно раскрыть через /guess @username твоя догадка.”

    С момента выбора квеста другие игроки могут пытаться угадать его задание.

    Игрок выполняет квест в общем чате до 20:00 по Москве.

    Бот сохраняет сообщения из общего чата.

    Другие игроки могут использовать команду:

    /guess @username текст догадки

    Если кто-то правильно угадывает задание, квест считается раскрытым:

    угадавший получает награду за этот квест;

    исполнитель квеста уже не может получить награду;

    исполнитель не получает штраф, потому что квест был раскрыт, а не провален.

    В 20:00 по Москве бот проверяет все активные квесты.

    За выполненный и нераскрытый квест игрок получает DUB.

    За невыполненный квест игрок теряет DUB.

    После проверки бот публикует итоги дня в общий чат.

    

    
    
    3. Почему нужно оповещать чат о выборе квеста

    Когда игрок выбирает квест, об этом нужно писать в общий чат.

    Это важно по трём причинам:

    Остальные игроки понимают, что с этого момента его можно раскрывать.

    Не возникает споров, можно ли было угадывать раньше.

    Это усиливает игру: все начинают наблюдать за поведением игрока после публичного объявления.

    Но уведомление должно быть аккуратным.

    Показывать можно:

    что игрок принял квест;

    что теперь его можно угадывать;

    команду для угадывания.

    Показывать нельзя:

    сложность квеста;

    награду;

    текст квеста;

    цель квеста;

    target-игрока;

    точный множитель награды.

    Пример хорошего сообщения:

    “🎭 @ivan принял секретный квест. Теперь его можно раскрыть через /guess @ivan твоя догадка.”

    Пример плохого сообщения:

    “@ivan выбрал тяжёлый квест на 600 DUB с целью @max.”

    Так делать нельзя, потому что это слишком много раскрывает.

    

    
    
    4. Основные технологии

    Использовать:

    Telegram Bot API;

    OpenAI API с недорогой моделью;

    базу данных;

    удалённый сервер;

    Docker / Docker Compose;

    простой скрипт для обновления бота на сервере;

    админ-команды в Telegram.

    OpenAI использовать только для смысловой проверки, а не для каждого сообщения.

    OpenAI нужен в двух местах:

    Проверить выполнение сложного квеста, если обычными правилами это сложно понять.

    Проверить, правильно ли игрок угадал чужой квест через /guess.

    

    
    
    5. Важные условия Telegram

    Бот должен быть добавлен в игровой групповой чат.

    Бот должен видеть сообщения группы. Если бот не видит обычные сообщения, нужно отключить privacy mode через BotFather или настроить бота так, чтобы он получал все нужные сообщения.

    Каждый игрок должен открыть личку с ботом и нажать /start, иначе бот не сможет присылать ему тайные квесты.

    Для функции mute бот должен быть администратором в группе и иметь право ограничивать участников.

    

    
    
    6. Роли

    
    Игрок

    Обычный участник игры.

    Может:

    подключиться к игре;

    смотреть свой кошелёк;

    смотреть лидерборд;

    получать квесты;

    выбирать квест;

    выполнять квест;

    угадывать чужие квесты;

    покупать предметы в магазине.

    
    
    Админ

    Пользователь, чей Telegram user id добавлен в список админов.

    Может:

    смотреть статус бота;

    вручную выдавать квесты;

    вручную запускать проверку;

    начислять и списывать DUB;

    перезагружать файл квестов;

    смотреть ошибки;

    делать объявления;

    управлять ботом на сервере, если это реализовано.

    

    
    
    
    7. Команды игрока

    
    /join

    Подключает игрока к игре.

    Если игрок новый:

    создать игрока в базе;

    создать кошелёк DUB;

    начислить стартовый баланс, если он задан;

    попросить открыть личку с ботом.

    Ответ в общем чате:

    “Ты в игре. Чтобы получать тайные квесты, открой личку с ботом и нажми /start.”

    Если игрок уже зарегистрирован:

    “Ты уже в игре.”

    

    
    
    /start в личке с ботом

    Сохраняет личный chat id игрока.

    Это нужно, чтобы бот мог отправлять ему квесты в личку.

    Ответ:

    “Личка подключена. Теперь я смогу присылать тебе тайные квесты.”

    Если игрок ещё не сделал /join в общем чате:

    “Личка подключена. Теперь напиши /join в игровом чате.”

    

    
    
    /wallet

    Показывает кошелёк игрока.

    Пример:

    “💰 Твой кошелёк:

    Баланс: 1250 DUBАктивные щиты: 2Double сегодня: не активенСегодня: +300 DUB / -100 DUB”

    Команда показывает только данные игрока, который её вызвал.

    

    
    
    /leaderboard

    Показывает рейтинг игроков по количеству DUB.

    Пример:

    “🏆 Лидерборд DUB

    @ivan — 3200 DUB

    @max — 2700 DUB

    @katya — 2100 DUB”

    Показывать топ-10.

    Если игрок не попал в топ-10, можно дополнительно показать его место:

    “Твоё место: 14 из 28.”

    

    
    
    /guess @user текст догадки

    Позволяет попытаться угадать чужой активный квест.

    Пример:

    /guess @ivan он должен был ответить кому-то реплаем и назвать его лохом

    Правила:

    нельзя угадывать свой квест;

    нельзя угадывать игрока, который ещё не выбрал квест;

    нельзя угадывать игрока, который не участвует в игре;

    нельзя угадывать квест, который уже раскрыт;

    нельзя угадывать после 20:00;

    первый правильно угадавший забирает награду за квест;

    исполнитель раскрытого квеста не получает награду;

    исполнитель раскрытого квеста не получает штраф;

    если догадка неправильная, квест остаётся активным.

    Проверка догадки делается через OpenAI.

    OpenAI должен сравнить:

    реальное задание;

    публичное описание смысла задания;

    текст догадки игрока.

    OpenAI должен вернуть строгий JSON:

    {  &quot;guessed&quot;: true,  &quot;confidence&quot;: 0.87,  &quot;reason&quot;: &quot;Догадка совпадает по смыслу: игрок должен был ответить реплаем и использовать заданное слово.&quot;}

    Если:

    guessed = true

    и

    confidence &gt;= 0.75

    то квест считается раскрытым.

    Ответ при успехе:

    “🕵️ @max раскрыл квест @ivan и забирает 300 DUB!”

    Ответ при неудаче:

    “Не похоже. Квест @ivan пока не раскрыт.”

    

    
    
    /shop

    Показывает магазин.

    Пример:

    “🛒 Магазин DUB

    /buy reroll — 150 DUBПоменять 3 предложенных квеста. Работает только до выбора квеста.

    /buy hint @user — 300 DUBКупить туманную подсказку на чужой квест.

    /buy mute @user — 1000 DUBЗамутить игрока на 5 минут. Shield может защитить.

    /buy shield — 700 DUBКупить щит от следующего mute.

    /buy double — 500 DUBУдвоить награду или штраф за сегодняшний квест.

    /buy jackpot 100Вложить DUB в общий банк дня.”

    

    
    
    
    8. Квесты

    Квесты курируются вручную.

    На первом этапе хранить квесты в JSON-файле.

    Админ должен иметь возможность перезагрузить квесты командой:

    /admin_reload_quests

    без полного перезапуска бота.

    

    
    
    9. Структура квеста

    Каждый квест должен содержать:

    id;

    включён или выключен;

    сложность;

    базовая награда;

    штраф;

    нужен ли target-игрок;

    как выбирать target-игрока;

    текст задания для игрока;

    публичное описание смысла задания для проверки /guess;

    правила простой проверки;

    нужно ли использовать OpenAI для проверки выполнения.

    Пример простого квеста:

    {  &quot;id&quot;: &quot;reply_call_loser&quot;,  &quot;enabled&quot;: true,  &quot;difficulty&quot;: &quot;medium&quot;,  &quot;base_reward&quot;: 250,  &quot;base_penalty&quot;: 100,  &quot;requires_target&quot;: true,  &quot;target_type&quot;: &quot;random_active_player&quot;,  &quot;title&quot;: &quot;Реплай с заданным словом&quot;,  &quot;description_template&quot;: &quot;Ответь реплаем на сообщение {target_username} фразой, в которой есть слово «лох».&quot;,  &quot;public_solution&quot;: &quot;Игрок должен был ответить реплаем конкретному игроку и использовать слово «лох».&quot;,  &quot;validation&quot;: {    &quot;type&quot;: &quot;reply_to_target_contains_text&quot;,    &quot;must_be_reply&quot;: true,    &quot;text_contains_any&quot;: [&quot;лох&quot;],    &quot;target_required&quot;: true  },  &quot;ai_validation&quot;: {    &quot;enabled&quot;: false  }}

    Пример сложного квеста:

    {  &quot;id&quot;: &quot;make_target_say_keyword&quot;,  &quot;enabled&quot;: true,  &quot;difficulty&quot;: &quot;hard&quot;,  &quot;base_reward&quot;: 600,  &quot;base_penalty&quot;: 250,  &quot;requires_target&quot;: true,  &quot;target_type&quot;: &quot;random_active_player&quot;,  &quot;title&quot;: &quot;Заставь игрока сказать слово&quot;,  &quot;description_template&quot;: &quot;Сделай так, чтобы {target_username} написал слово «пельмени». Сам не пиши это слово первым.&quot;,  &quot;public_solution&quot;: &quot;Игрок должен был спровоцировать конкретного участника написать слово «пельмени», не написав его первым.&quot;,  &quot;validation&quot;: {    &quot;type&quot;: &quot;custom_ai_check&quot;  },  &quot;ai_validation&quot;: {    &quot;enabled&quot;: true  }}

    

    
    
    10. Сложности квестов

    У квестов должно быть 3 уровня сложности.

    
    Лёгкий

    Простые действия.

    Примеры:

    написать сообщение с нужным словом;

    получить реакцию;

    ответить кому-то реплаем;

    заставить кого-то ответить короткой фразой.

    Пример награды:

    100 DUB.

    Пример штрафа:

    50 DUB.

    

    
    
    Средний

    Требует более заметного, но не слишком сложного действия.

    Примеры:

    ответить конкретному игроку реплаем с нужным словом;

    заставить другого игрока написать определённое слово;

    получить ответ от target-игрока;

    начать обсуждение заданной темы.

    Пример награды:

    250 DUB.

    Пример штрафа:

    100 DUB.

    

    
    
    Тяжёлый

    Требует более хитрого поведения.

    Примеры:

    заставить конкретного игрока написать конкретное слово;

    спровоцировать цепочку из нескольких сообщений;

    добиться, чтобы несколько игроков ответили на одну тему;

    выполнить действие, которое сложно проверить без смысла контекста.

    Пример награды:

    600 DUB.

    Пример штрафа:

    250 DUB.

    

    
    
    
    11. Выдача квестов в 08:00

    Каждый день в 08:00 по Москве бот берёт всех активных игроков.

    Для каждого игрока:

    Проверяет, есть ли личный chat id.

    Если личка не подключена, квесты не отправляются.

    Выбирает 3 квеста:

    1 лёгкий;

    1 средний;

    1 тяжёлый.

    Если квест требует target-игрока, бот выбирает случайного активного игрока, кроме самого исполнителя.

    Отправляет игроку 3 варианта в личку.

    Добавляет кнопки выбора.

    Пример сообщения в личке:

    “🎲 Выбери тайный квест на сегодня:

    Лёгкий — 100 DUBСделай так, чтобы кто-то ответил тебе словом ‘согласен’.

    Средний — 250 DUBОтветь реплаем на сообщение @max фразой, где есть слово ‘лох’.

    Тяжёлый — 600 DUBСделай так, чтобы @katya написала слово ‘пельмени’, но сам не пиши это слово первым.

    Чем раньше возьмёшь квест, тем выше награда.”

    Кнопки:

    “Взять лёгкий”

    “Взять средний”

    “Взять тяжёлый”

    

    
    
    12. Выбор квеста кнопкой

    Когда игрок нажимает кнопку выбора квеста:

    Бот сохраняет выбранный квест.

    Сохраняет точное время выбора.

    Считает модификатор награды по времени.

    Считает возможную итоговую награду.

    Отправляет игроку подтверждение в личку.

    Пишет в общий чат, что игрок принял секретный квест.

    Ответ игроку в личку:

    “Квест принят.

    Текущий множитель награды: x1.82Возможная награда: 455 DUBПроверка будет в 20:00 по Москве.

    Не спались.”

    Сообщение в общий чат:

    “🎭 @ivan принял секретный квест. Теперь его можно раскрыть через /guess @ivan твоя догадка.”

    Важно:

    не писать в общий чат сложность;

    не писать награду;

    не писать target-игрока;

    не писать текст квеста;

    не писать подсказку;

    не писать множитель.

    

    
    
    13. Когда можно угадывать квест

    Угадывать квест можно только после того, как игрок выбрал задание.

    То есть состояние игрока за день может быть таким:

    no_quest_offered — квесты ещё не предложены.

    offered — 3 квеста отправлены в личку, но игрок ещё не выбрал.

    accepted — игрок выбрал квест, его уже можно угадывать.

    revealed — квест раскрыт другим игроком.

    completed — квест выполнен и проверен.

    failed — квест провален.

    expired — день закончился.

    Команда /guess должна работать только в состоянии accepted.

    Если игрок пытается угадать квест, который ещё не принят:

    “Этот игрок ещё не принял секретный квест. Пока угадывать нечего.”

    Если квест уже раскрыт:

    “Этот квест уже раскрыт.”

    Если проверка дня уже прошла:

    “Сегодняшняя охота уже закончилась.”

    

    
    
    14. Модификатор награды по времени

    Игрок может принять квест с 08:00 до 20:00 по Москве.

    Чем раньше игрок принимает квест, тем выше итоговая награда.

    Правила:

    в 08:00 множитель x2.0;

    к 20:00 множитель падает до x0.1;

    минимальный множитель x0.1.

    Формула:

    доля_дня = (время_принятия - 08:00) / 12 часовмножитель = 2.0 - доля_дня * 1.9

    Примеры:

    08:00 → x2.011:00 → примерно x1.52514:00 → примерно x1.0517:00 → примерно x0.57519:50 → примерно x0.12620:00 → x0.1

    Итоговая награда:

    итоговая_награда = базовая_награда * множитель

    Округлять до целого DUB.

    Если игрок купил double, итоговая награда умножается ещё на 2.

    

    
    
    15. Хранение сообщений

    Бот должен сохранять сообщения из общего чата.

    Важно: вечером бот не должен пытаться “достать историю чата из Telegram”. Он должен сохранять сообщения сразу, когда они появляются.

    Для каждого сообщения сохранять:

    Telegram message id;

    chat id;

    user id автора;

    username автора;

    текст;

    время;

    тип сообщения;

    является ли сообщение reply;

    id сообщения, на которое ответили;

    user id автора сообщения, на которое ответили;

    упоминания пользователей, если есть.

    Для MVP достаточно хорошо сохранять:

    текст;

    автора;

    время;

    reply-связи;

    id сообщения.

    Сообщения нужны для проверки квестов.

    

    
    
    16. Простые проверки без OpenAI

    Часть квестов нужно проверять обычными правилами без OpenAI.

    
    message_contains_text

    Игрок написал сообщение с нужным словом.

    Проверять:

    сообщение после времени принятия квеста;

    автор — исполнитель;

    текст содержит нужное слово.

    

    
    
    reply_contains_text

    Игрок ответил реплаем и написал нужное слово.

    Проверять:

    сообщение после времени принятия квеста;

    автор — исполнитель;

    сообщение является reply;

    текст содержит нужное слово.

    

    
    
    reply_to_target_contains_text

    Игрок ответил реплаем на сообщение конкретного target-игрока и написал нужное слово.

    Проверять:

    сообщение после времени принятия квеста;

    автор — исполнитель;

    сообщение является reply;

    автор исходного сообщения — target;

    текст содержит нужное слово.

    

    
    
    target_says_keyword_after_player_message

    Target-игрок написал нужное слово после сообщения исполнителя.

    Проверять:

    target написал keyword после времени принятия квеста;

    target написал keyword после сообщения исполнителя;

    исполнитель не написал keyword первым, если это запрещено заданием;

    сообщение target было до 20:00.

    

    
    
    custom_ai_check

    Проверка через OpenAI.

    Используется, когда простыми правилами нельзя уверенно понять, выполнен ли квест.

    

    
    
    
    17. OpenAI-проверка выполнения квеста

    OpenAI использовать только для квестов, где:

    &quot;ai_validation&quot;: {  &quot;enabled&quot;: true}

    Не отправлять в OpenAI весь чат за день.

    Отправлять только нужный контекст:

    описание квеста;

    публичное описание смысла квеста;

    имя исполнителя;

    target-игрока, если есть;

    сообщения исполнителя после принятия квеста;

    сообщения target-игрока, если target есть;

    несколько сообщений вокруг возможного выполнения;

    время принятия квеста;

    время проверки.

    OpenAI должен вернуть строгий JSON:

    {  &quot;completed&quot;: true,  &quot;confidence&quot;: 0.92,  &quot;reason&quot;: &quot;Игрок задал наводящий вопрос, после чего целевой игрок написал нужное слово.&quot;}

    Если:

    completed = true

    и

    confidence &gt;= 0.70

    то квест считать выполненным.

    Если confidence ниже 0.70, считать квест невыполненным или отправлять на ручную проверку админу.

    Для MVP можно считать невыполненным.

    

    
    
    18. OpenAI-проверка /guess

    Когда игрок пишет:

    /guess @ivan он должен был заставить Макса написать слово пельмени

    бот отправляет в OpenAI:

    реальный квест @ivan;

    публичное описание смысла квеста;

    текст догадки;

    кто угадывает;

    чей квест угадывают.

    OpenAI должен вернуть строгий JSON:

    {  &quot;guessed&quot;: true,  &quot;confidence&quot;: 0.86,  &quot;reason&quot;: &quot;Догадка совпадает по смыслу: игрок должен был заставить конкретного участника написать слово.&quot;}

    Если:

    guessed = true

    и

    confidence &gt;= 0.75

    то:

    квест считается раскрытым;

    сохраняется, кто раскрыл;

    угадавший получает награду за квест;

    исполнитель квеста больше не может получить награду;

    исполнитель не получает штраф.

    Если догадка неправильная:

    ничего не менять;

    квест остаётся активным.

    

    
    
    19. Проверка квестов в 20:00

    Каждый день в 20:00 по Москве бот запускает проверку.

    Для каждого игрока:

    Если игрок не выбрал квест — ничего не делать.

    Если игрок выбрал квест, но квест был раскрыт — исполнитель ничего не получает и не теряет.

    Если квест не раскрыт — проверить выполнение.

    Если выполнен — начислить DUB.

    Если не выполнен — списать штраф.

    Если активен double:

    награда удваивается;

    штраф удваивается;

    если квест был раскрыт, double сгорает без эффекта.

    Записать результат в базу.

    Добавить игрока в итоги дня.

    Пример итогового сообщения:

    “🌙 Итоги тайных квестов

    ✅ @ivan выполнил квест: +430 DUB🕵️ @max раскрыл квест @sergey и забрал 250 DUB❌ @oleg не выполнил квест: -100 DUB🎰 Jackpot 700 DUB забирает @katya

    Топ дня: 1. @ivan: +4302. @max: +2503. @oleg: -100”

    

    
    
    20. Кошелёк DUB

    У каждого игрока есть баланс DUB.

    Все изменения баланса должны записываться отдельными операциями.

    Типы операций:

    стартовый бонус;

    награда за квест;

    штраф за невыполненный квест;

    награда за раскрытие чужого квеста;

    покупка в магазине;

    вклад в jackpot;

    выигрыш jackpot;

    админское начисление;

    админское списание;

    возврат из-за ошибки.

    Важно:

    нельзя менять баланс без записи причины;

    нельзя разрешать покупку, если у игрока недостаточно DUB;

    баланс может уходить в минус или не может — это должно быть настройкой.

    Для MVP лучше не разрешать уходить в минус.

    

    
    
    21. Магазин DUB

    Команда:

    /shop

    Показывает список покупок.

    Покупка:

    /buy название

    или:

    /buy название @user

    

    
    
    22. Reroll

    Команда:

    /buy reroll

    Цена:

    150 DUB.

    Эффект:

    игрок получает новые 3 квеста на выбор;

    работает только до принятия квеста;

    старые предложенные квесты заменяются новыми.

    Если игрок уже выбрал квест:

    “Ты уже выбрал квест на сегодня. Reroll больше недоступен.”

    

    
    
    23. Hint

    Команда:

    /buy hint @user

    Цена:

    300 DUB.

    Эффект:

    бот даёт туманную подсказку по квесту другого игрока.

    Подсказка должна помогать, но не раскрывать квест полностью.

    Примеры подсказок:

    “Квест связан с reply.”

    “Квест связан с конкретным словом.”

    “Квест связан с другим игроком.”

    “Квест требует спровоцировать реакцию.”

    “Квест связан с сообщением после ответа другого игрока.”

    Если у цели нет активного квеста:

    “У этого игрока сейчас нет активного квеста.”

    Если квест уже раскрыт:

    “Этот квест уже раскрыт.”

    

    
    
    24. Mute

    Команда:

    /buy mute @user

    Цена:

    1000 DUB.

    Эффект:

    бот ограничивает игроку возможность писать в чат на 5 минут.

    Условия:

    бот должен быть админом;

    нельзя мутить себя;

    нельзя мутить бота;

    нельзя мутить админов;

    нельзя мутить игрока, который не участвует в игре;

    желательно нельзя мутить одного и того же игрока чаще 1 раза в час.

    Shield-логика:

    если у цели есть хотя бы один shield, mute не срабатывает;

    у цели сгорает один shield;

    покупатель всё равно тратит DUB;

    в чат пишется:

    “🛡 @ivan попытался замутить @max, но щит спас @max. Один shield сгорел.”

    Если shield нет:

    “🔇 @ivan купил mute для @max на 5 минут.”

    Если бот не может выполнить mute из-за отсутствия прав:

    DUB не списывать;

    написать покупателю:

    “Я не могу замутить игрока: у меня нет нужных прав администратора.”

    

    
    
    25. Shield

    Команда:

    /buy shield

    Цена:

    700 DUB.

    Эффект:

    добавляет игроку один shield;

    shield защищает от следующего mute;

    щитов может быть сколько угодно;

    один входящий mute сжигает один shield.

    Пример ответа:

    “🛡 Shield куплен. Активных щитов: 3.”

    

    
    
    26. Double

    Команда:

    /buy double

    Цена:

    500 DUB.

    Эффект:

    удваивает награду за сегодняшний квест;

    если квест не выполнен, штраф тоже удваивается;

    если квест раскрыли, double сгорает без пользы;

    купить можно только после принятия квеста и до 20:00;

    купить можно только один double в день.

    Пример:

    обычная награда 400 DUB;

    с double игрок получит 800 DUB.

    Если штраф 100 DUB:

    с double игрок потеряет 200 DUB.

    После покупки можно написать в общий чат или только в личку.

    Лучше в общий чат:

    “🎲 @ivan активировал double на сегодняшний квест. Теперь он играет по-крупному.”

    Это не раскрывает квест, но добавляет давления.

    

    
    
    27. Jackpot

    Команда:

    /buy jackpot 100

    Эффект:

    игрок кладёт указанную сумму DUB в общий банк дня.

    Правила:

    минимальный вклад: 50 DUB;

    максимальный вклад от одного игрока за день: 500 DUB;

    в 20:00 jackpot получает случайный игрок из тех, кто:

    выбрал квест;

    выполнил квест;

    не был раскрыт.

    Если никто не выполнил квест:

    jackpot переносится на следующий день.

    Пример сообщения:

    “🎰 @ivan добавил 100 DUB в jackpot. Банк дня: 750 DUB.”

    34. Поведение при ошибках

    
    Игрок не открыл личку

    Если игрок сделал /join, но не открыл личку:

    не отправлять ему квесты;

    в общий чат можно написать напоминание:

    “@ivan, чтобы получать тайные квесты, открой личку с ботом и нажми /start.”

    

    
    
    OpenAI недоступен

    Если OpenAI недоступен:

    простые квесты проверять как обычно;

    сложные квесты помечать как “требует ручной проверки”;

    уведомить админа.

    Для MVP можно считать такие квесты невыполненными, но лучше помечать их на ручную проверку.

    

    
    
    Бот не может замутить игрока

    Если бот не имеет прав:

    не списывать DUB;

    написать игроку, что покупка не выполнена.

    

    
    
    Игрок сменил username

    Лучше хранить игроков по Telegram user id, а не по username.

    Username использовать только для отображения.

    

    
    
    У игрока нет username

    Команда /guess @user может не работать, если у человека нет username.

    Поэтому желательно поддержать альтернативу:

    угадывать через reply на сообщение игрока;

    или через список игроков;

    или через внутренний id.

    Для MVP можно оставить username, но лучше заложить поддержку reply.

    Пример:

    Игрок отвечает реплаем на сообщение @ivan:

    /guess он должен был заставить кого-то написать пельмени

    Бот понимает, что цель guess — автор сообщения, на которое сделан reply.

    36. Критерии готовности

    Бот считается готовым, если:

    Игрок может подключиться через /join.

    Игрок может подключить личку через /start.

    В 08:00 бот присылает 3 квеста.

    Игрок выбирает квест кнопкой.

    После выбора бот пишет в общий чат, что игрок принял секретный квест.

    До выбора квеста игрока нельзя угадывать.

    После выбора квеста игрока можно угадывать через /guess.

    Бот сохраняет сообщения группы.

    /wallet показывает правильный баланс.

    /leaderboard показывает рейтинг.

    /guess может раскрыть чужой квест.

    Простые квесты проверяются без OpenAI.

    Сложные квесты проверяются через OpenAI.

    В 20:00 бот начисляет награды и штрафы.

    Reroll работает.

    Hint работает.

    Mute работает, если бот админ.

    Shield защищает от mute и сгорает по одному.

    Double удваивает награду или штраф.

    Jackpot принимает DUB и разыгрывается.

    Админ может перезагрузить квесты.

    Админ может вручную запустить проверку.

    Бот можно обновить на сервере одной командой.

    После перезапуска данные не теряются.

    В логах видно, что происходило с квестами, DUB и покупками.
